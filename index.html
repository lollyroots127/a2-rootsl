Web Dev Concepts

This file is named index.html, and it is the designated homepage of the /~rootsl/a1-rootsl/ directory located on the OSU Engineering web server. When a web browser sends a Hypertext Transfer Protocol (HTTP) request for the directory, the Apache web server, the software used by OSU Engineering, maps the request to this file. The majority of web servers are configured to search for an index.html file whenever they receive a request for /, the root directory. On some web servers, other file names are accepted as homepages. For example, Microsoft web servers will also look for default.html files.

The Inspection tools in web browsers such as Firefox and Google Chrome allow users to view requests and responses between the web client and different servers. The network information can differ based on which protocol is used. When viewing the local file version of this webpage, the network information will show three resources: index.html, main.css, and main.js. The general network information for the index.html file shows the Universal Resource Locator (URL), request method, status code, and referrer policy. It shows limited response and request headers as well. When the same file is accessed through a web server, the network information is quite different. An additional favicon.ico file is shown. The general tab includes a remote Internet Protocol (IP) address of where the server is located. The request and response headers are more detailed, showing information on the server, the user-agent, and time and date signatures.

When viewing the network tab for the file hosted on the OSU Engineering web server, four main resources are listed with different status codes. The index.html file and favicon.ico files have 200 status codes, meaning the files were found and retrieved successfully. The main.css and main.jss files are listed with 404 status codes, meaning the files were not found. The only file located in the /~rootsl/a1-rootsl/ directory was index.html. The main.css and main.jss files are linked in the index.html file, but not provided or located on the web server. Upon inspection, the URL for the favicon.ico file is located inside the root directory of the web.engr.oregonstate.edu server. It is a file provided by the parent directory, therefore it has a success status code.

The URL of this webpage can be broken into four main components: the scheme, the subdomain, the host domain, and the resource. The scheme in this case is https, which means the web browser used a Secure HTTP protocol to access the requested resource. The subdomains are web and engr. These identify what part of the server the request should be sent to. The host domain or server name is oregonstate.edu. The server name identifies which machine, or server, the resource is hosted in. The Domain Name System (DNS) maps the server name to a machine-understandable IP address. Finally, the path to resource is /~rootsl/a1-rootsl/. This identifies where the server should look to find the resource.
